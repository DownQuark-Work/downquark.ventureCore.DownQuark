Design Systems

about

DownQuark utilizes multiple design systems for the various projects and tasks we produce. There will be overlapping elements throughout the systems as we continue building. The design system itself will help to handle and maintain much of this overlap using various methods that will be easy on the designer and/or developer to implement.

The specifics will be defined on the individual components elements within the specific system.

namespacing

To assist in organizing the different Design Systems we utilize namespacing. This allows us to categorically define and display the correct styles in an intuitive fashion.

inheritence

As mentioned above, we plan on introducing a unique aspect within our separate Design Systems. One that involves a limited, opinionated hierarchy.

The end goal is that this will allow us to have DRY components, that once specified in various systems can be re-used without any extra overhead in as many locations as desired.

While the nuts and bolts have not been finalized, this is one of the tasks we are most excited about building.
