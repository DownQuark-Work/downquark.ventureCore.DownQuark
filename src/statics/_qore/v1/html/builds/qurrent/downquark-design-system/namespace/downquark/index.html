namespace: downquark

more of a styleguide?

It could be argued that nothing in this layer is actually a Design System, but it provides a stable foundation none-the-less.

Everything contained within this namespace can be utilized in all of the other Design Systems without fear of conflicts.

Everything from this layer is also created to be overwritten. It’s the base to use when starting something from scratch. Or an include to fill in some missing gaps.

The following are contained in this layer:

Quark Design Tokens

These are the most basic abstractions of our Design Tokens.

Naming is straightforward, CTI is not yet in effect, and everything is is native to css.
Stylesheet imports, clamp, and nesting are allowed. But custom fonts or any values needing a pre/post-processor are not.

Classless Stylesheets

What styles a lot of our more static pages. Replacing convoluted components with css queries.
(viewable in base html pages)
